## 1. MiMo Token Plan 深度分析（14:21-15:24）

**参与者：** ou_0dc1627aab8e3161564ca8630b4e8f07、ou_3927a1cc20b59bdbaeddc01926103ec7、小瓜

**话题：** MiMo Token Plan 定价策略 vs MiniMax Token Plan

### 核心讨论内容

**MiniMax Token Plan 定价（官网数据）：**
- Starter ¥29/月 → 20M tokens / 每5小时
- Plus ¥49/月 → 50M tokens / 每5小时
- Max ¥119/月 → 150M tokens / 每5小时
- 旗舰 ¥199/月 → 不限 tokens，4500次调用 / 每5小时

**MiMo Token Plan 定价：**
- Lite ¥39/月 → 60M Credits
- Standard ¥99/月 → 200M Credits
- Pro ¥329/月 → 700M Credits
- Max ¥659/月 → 1600M Credits
- Credit 消耗：Omni 1x、Pro 256k 2x、Pro 1M 4x

### 关键发现

1. **消费群体主要是"养虾人"**（Agent 用户），不是普通聊天用户。Agent 任务 token 消耗是普通聊天的数十倍。

2. **MiMo 的计费方向是对的** — 无时间窗口、集中消耗，更适合 Agent 的爆发式使用模式。MiniMax 的5小时窗口可能中断长任务。

3. **但定价完全错误** — MiMo-V2-Pro 实际推理成本估计 $0.07-0.2/M tokens，API 定价 $1/M 是 5-15 倍加价，Token Plan 毛利 7-20 倍。有全行业最低的推理成本，却没有打出价格优势。

4. **内部利益冲突** — API 定价权在 AI 团队（激进低价），Token Plan 定价权在平台团队（保守高价），两套体系互相打架。

5. **Credit 乘数是毒药** — Agent 用户最需要 Pro 模型，但 Pro ×2、长上下文 ×4 的设计让重度用户实际成本更高。

### 结论

MiMo 手里有王炸（极低推理成本 + MoE 架构），出错了牌（高定价 + Credit 乘数）。差的是临门一脚：取消乘数、降低 Token Plan 价格、推出 Agent 专项方案。
