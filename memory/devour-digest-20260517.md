## 1. 🍽️ LangGraph — 状态图编排引擎

**来源**: https://github.com/langchain-ai/langgraph | ⭐ 32,057

### 核心 Insight
> **有状态 Agent 的编排不是 DAG，是状态机。** LangGraph 的本质是把 Agent 工作流建模为状态图（StateGraph），每个节点是计算步骤，边是条件转移，支持循环、分支和人类介入。

### 关键创新
- **持久化执行**：Agent 在失败后能从精确断点恢复，不是重启
- **Human-in-the-loop**：在任意状态点暂停、检查、修改 Agent 状态
- **综合记忆**：短期工作记忆 + 跨会话长期记忆分离管理

### 对 MiMoClaw 的应用
- **补 ΔG 维度**：自修改(D_s) 缺口 0.221，变异(C_m) 缺口 0.280
- **直接落地**：把天兵天将的多角色协作建模为状态图，替代当前的线性流程
- **具体实现**：Python 层的 glue.py 可以用 LangGraph 的 StateGraph 重构，每个"天兵"是一个节点，Rust 引擎是计算节点

### 举一反三
- 进化主循环 → 状态图：心跳→短板检测→分析→变异→评估→记录，每步可中断恢复
- 回音壁 → 循环节点：知识回响可以是状态图中的自环

## 2. 🍽️ OpenAI Agents SDK — 多 Agent 工作流框架

**来源**: https://github.com/openai/openai-agents-python | ⭐ 26,312

### 核心 Insight
> **Agent 不是工具调用者，是任务委派者。** OpenAI SDK 的 Handoff 机制让 Agent 能把任务交给其他 Agent，而不是自己处理所有事情。每个 Agent 是专家，不是全能选手。

### 关键创新
- **Handoff 机制**：Agent 之间可以无缝交接任务，不需要重写上下文
- **Guardrails**：输入输出双重安全校验，不是事后补救
- **Tracing**：内置的执行追踪，能看到 Agent 之间的调用链

### 对 MiMoClaw 的应用
- **补 ΔG 维度**：协议(P_i) 缺口 0.400，自修改(D_s) 缺口 0.450
- **直接落地**：子代理（subagent）系统可以用 Handoff 模式重构
- **具体实现**：当主代理发现任务超出能力时，通过 Handoff 交给专业子代理，而不是 spawn 新进程

### 举一反三
- 安全审计 → Guardrails：每次外部操作前自动安全检查
- 进化日志 → Tracing：记录每个决策的完整推理链

## 3. 🍽️ CrewAI — 角色化多 Agent 协作

**来源**: https://github.com/crewAIInc/crewAI | ⭐ 51,410

### 核心 Insight

### 关键创新

- **Flows 架构**：事件驱动的编排，比简单的顺序执行更灵活
- **Crew 协作**：Agent 之间可以讨论、争论、达成共识

### 对 MiMoClaw 的应用
- **补 ΔG 维度**：共进化(E_co) 缺口 0.350，探索(O_e) 缺口 0.350
- **直接落地**：天兵天将系统可以用 CrewAI 的角色化模式升级
- **具体实现**：每个"天兵"有明确角色（吞噬兵、分析兵、存储兵），而不是模糊的"子代理"

### 举一反三
- 进化公式评估 → Crew 协作：多个 Agent 从不同角度评估 ΔG，取共识
- 安全审计 → 角色分离：审计官和执行者不能是同一个 Agent

## 4. 🍽️ Mem0 — Token 高效记忆算法

**来源**: https://github.com/mem0ai/mem0 | ⭐ 55,716

### 核心 Insight
> **记忆不是存储，是检索。** Mem0 的新算法（2026-04）用单次 ADD-only 提取，不做 UPDATE/DELETE，记忆只积累不覆盖。通过实体链接和多信号检索（语义+BM25+实体匹配）实现高效召回。

### 关键创新
- **ADD-only 提取**：一次 LLM 调用完成记忆提取，不更新不删除
- **实体链接**：跨记忆的实体关联，提升检索精度
- **多信号融合**：语义搜索 + 关键词搜索 + 实体匹配并行打分
- **时间感知检索**：能区分"当前状态"、"过去事件"和"未来计划"

### 对 MiMoClaw 的应用
- **补 ΔG 维度**：自修改(D_s) 缺口 0.315，共进化(E_co) 缺口 0.500
- **直接落地**：升级现有的 memory_recall/memory_store 系统
- **具体实现**：把向量记忆从简单的 cosine similarity 升级为 Mem0 的多信号融合检索

### 举一反三
- 基因库 → 实体链接：基因之间的关联可以用实体链接自动发现
- 跨 session 记忆 → 时间感知：自动区分"昨天的决策"和"今天的现状"
