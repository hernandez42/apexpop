## 三、探索引擎升级方案

### 3.1 子代理搜索结果带回上下文

**现状**: 子代理返回 `{"conclusion": "...", "score": 0.85}`
**目标**: 子代理返回完整的探索记录

```jsonl
{
  "explore_id": "exp_20260517_0643",
  "timestamp": "2026-05-17T06:43:00",
  "query": "搜索EvoAgentX的AFlow拓扑优化",
  "context": {
    "why": "O_e=0.49, 需要提升探索能力",
    "previous_attempts": ["2026-05-12 吞噬过EvoAgentX"],
    "knowledge_gap": "AFlow的拓扑优化具体怎么映射到我的进化系统"
  },
  "process": [
    {"step": "web_search", "query": "EvoAgentX AFlow topology optimization", "results": 3},
    {"step": "web_fetch", "url": "...", "extracted": 1200, "key_insight": "..."},
    {"step": "analysis", "reasoning": "AFlow用DAG表示工作流..."},
    {"step": "cross_domain", "applied_to": "MiMoClaw进化循环", "result": "..."}
  ],
  "conclusion": "AFlow拓扑优化可以映射到技能间连接方式",
  "confidence": 0.82,
  "tags": ["topology", "workflow", "dag", "optimization"],
  "verification": {
    "tested": false,
    "source_url": "https://...",
    "source_quality": "ICLR 2026 论文"
  }
}
```

**实施**: 修改子代理搜索逻辑，要求返回 `context` + `process` + `tags` 字段。

### 3.2 知识索引系统 (Signal-Tag Mapping)

**设计**: 为每条知识添加信号标签，建立可检索的知识网络

```jsonl
{
  "id": "k001",
  "domain": "哲学",
  "topic": "实践论",
  "insight": "实践是检验真理的唯一标准",
  "source": "毛泽东选集",
  "confidence": 0.95,
  "tags": ["验证", "闭环", "反馈", "真理", "实践"],
  "signals": {
    "triggers": ["需要验证假设时", "闭环不完整时"],
    "inhibits": ["纯理论推导时"],
    "amplifies": ["与其他验证方法叠加"]
  },
  "connections": ["k007 知行合一", "k003 APEX公式验证"],
  "applied_count": 3,
  "last_applied": "2026-05-17",
  "timestamp": "2026-05-12"
}
```

**知识索引结构**:
```
knowledge-index.jsonl
├── tags → [knowledge_ids]         // 标签到知识的映射
├── signals → [knowledge_ids]      // 信号到知识的映射
├── connections → [knowledge_id]    // 知识之间的关联
└── gaps → [domain + signal]        // 知识空白区域
```

### 3.3 探索-验证-沉淀闭环

```
┌─────────────────────────────────────────────────┐
│              探索引擎闭环 (Explore-Verify-Settle)  │
│                                                   │
│  ① 探索 (Explore)                                │
│  ├─ 识别知识空白 (gaps)                            │
│  ├─ 生成探索问题                                   │
│  ├─ 派遣子代理 (带上下文)                          │
│  └─ 收集完整探索记录                               │
│                                                   │
│  ② 验证 (Verify)                                 │
│  ├─ 交叉验证: 与已有知识对比                        │
│  ├─ 实践验证: 是否能解决真实问题                    │
│  ├─ 一致性验证: 是否与核心原则冲突                  │
│  └─ 评估置信度                                    │
│                                                   │
│  ③ 沉淀 (Settle)                                 │
│  ├─ 写入 knowledge-base.jsonl (带 tags/signals)   │
│  ├─ 更新 knowledge-index.jsonl (索引映射)          │
│  ├─ 建立 connections (知识关联)                    │
│  ├─ 更新 reflection-genes (反思链)                │
│  └─ 触发 ΔG 重新评估                              │
│                                                   │
│  ④ 反馈 (Feedback)                               │
│  ├─ 本次探索的 ΔO_e 增量                          │
│  ├─ 下次探索的优先级调整                           │
│  └─ 知识空白区域更新                               │
└─────────────────────────────────────────────────┘
```
