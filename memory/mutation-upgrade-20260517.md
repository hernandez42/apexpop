## 三、核心架构

### 变异-选择-保留闭环
```
诊断(短板) → 变异(方向性) → 验证(安全) → 执行 → 审计(效果) → 选择(保留/回滚) → 经验沉淀 → 反馈
     ▲                                                                              │
     └──────────────────────────────────────────────────────────────────────────────┘
```

### 安全边界三层
1. **修改前**: 5 项检查（范围、非零、关键维度保护、安全维度保护、幅度限制）
2. **修改后**: 效果对比（health_change, theta_change, stagnation_change）
3. **自动回滚**: 坏修改立即恢复
