## 二、关键变更

### 变更 1：通信模式 — 广播 → 共振

| 维度 | 旧（广播） | 新（共振） |
|------|-----------|-----------|
| 方向 | 单向 A→B | 双向 A↔B |
| 时机 | 异步 | 同步窗口期（秒级） |
| 响应 | 被动读取 | 主动响应 |
| 冲突 | 无检测 | 冲突检测+仲裁 |
| 历史 | 无记录 | 交互矩阵记录 |

**核心改造**：`shared_experience.jsonl` → `ResonanceBus`（共振总线）
- 事件发射 → 路由匹配 → 窗口期响应 → 冲突检测 → 决策合成

### 变更 2：角色关系 — 合作 → 对抗

| 旧角色 | 新角色 | 对标 MAE |
|--------|--------|----------|
| Core1: 服务交付 | Core1: **需求 Proposer** | Proposer |
| Core3: 进化引擎 | Core3: **方案 Solver** | Solver |
| Core2: 安全审计 | Core2: **质量 Judge** | Judge |

**关键转变**：
- 旧：三核各自独立进化，通过广播分享经验
- 新：三核形成对抗闭环 — Proposer 挑战 Solver，Judge 裁决
- Proposer 奖励 = Solver 失败率（军备竞赛）
- Judge 一票否决权（安全维度）

### 变更 3：新增交互矩阵 Σ

从 Self-Play 论文引入的核心数据结构：

```
记录内容：
- 谁对谁做了什么（交互类型）
- 结果如何（通过/否决/修订）
- ΔG 影响（实际效果）
- 时间戳（时序分析）

用途：
- MSS 元策略求解器（推荐最优进化策略）
- 瓶颈识别（哪种交互模式导致停滞）
- 否决模式分析（减少重复否决）
- 非传递性检测（石头剪刀布关系）
```

### 变更 4：新增进化法官

从 MAE 的 Judge 机制工程化：

```
评估维度：
1. 必要性 (necessity) — 0.25 权重
2. 可行性 (feasibility) — 0.20 权重
3. 安全性 (safety) — 0.25 权重（一票否决）
4. 原创性 (originality) — 0.15 权重
5. 影响力 (impact) — 0.15 权重

质量门控：
- 安全分 < 0.7 → 一票否决
- 总分 < 0.6 → 否决
- 全部通过 → 进入能力库

自校准：
- 统计评估与实际效果的相关性
- 准确度低的维度降低权重
```

### 变更 5：策略种群 + 元策略求解器

从 Self-Play 综述引入：

```
4 种进化策略：
- 激进策略：ΔG优先=0.8, 安全放松=0.3
- 保守策略：ΔG优先=0.3, 安全放松=0.0
- 探索策略：ΔG优先=0.5, 安全放松=0.2
- 修复策略：ΔG优先=0.6, 安全放松=0.1

选择逻辑（MSS）：
- 停滞 → 探索
- 连续否决 → 保守
- 新需求 → 激进
- 有 bug → 修复
```

### 变更 6：E_co 维度公式重定义

```
旧：E_co = 共享次数 / 总操作次数（太简单）

新：E_co = α × 共振成功率 
         + β × 交互矩阵覆盖率 
         + γ × 法官通过率 
         + δ × 对抗进化强度

四项指标各有含义，比单一共享率更能反映共进化质量。
```

## 五、吞噬结论

### 真正消化了什么

1. **共进化 = 对抗 + 裁判 + 同步**，不是"大家一起变好"
2. **交互矩阵是共进化的记忆**，没有记忆的共进化就是盲目碰撞
