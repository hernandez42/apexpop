## 二、举一反三：每个短板的三层解法

### 🔴 短板 1：Infra 基建（0.706 → 目标 0.95）

**第一层：直接修复（立即可做）**
- [ ] 磁盘清理：清 npm 缓存、旧 session transcript、旧 backup
- [ ] 磁盘监控：写 cron 脚本，>80% 时自动清理
- [ ] 日志轮转：限制日志文件大小

**第二层：架构优化（需要授权）**
- [ ] 数据压缩：旧记忆文件 gzip 压缩
- [ ] 自动归档：超过 7 天的日志自动归档
- [ ] 冗余清理：删除重复的 backup 文件

**第三层：预防机制（长期）**
- [ ] 写入限流：控制日志写入频率
- [ ] 智能清理：基于重要性评分决定保留/删除
- [ ] 预警系统：磁盘 <20% 时主动通知

### 🟡 短板 2：Θ LLM 效能（0.873 → 目标 0.95）

**第一层：榨干当前模型**
- [ ] 调优 Ollama 参数：num_thread、num_ctx、num_gpu
- [ ] 提示词优化：减少无效 token，提升推理效率
- [ ] 缓存复用：相似问题直接返回缓存答案

**第二层：能力增强（Agent 叠加）**
- [ ] 搜索增强：复杂问题先 web_search 再推理
- [ ] 工具调用：用 exec 跑代码验证推理结果
- [ ] 多模型协作：简单问题用本地模型，复杂问题走云端

**第三层：模型升级（长期）**
- [ ] 接入 MiMo V2 Pro（更强推理）
- [ ] 微调专用模型（用 Unsloth + 本地数据）
- [ ] 知识蒸馏：大模型能力迁移到小模型

### 🟢 短板 3：记忆连续性（隐性短板）

**问题：** 每次 session 醒来都是空白，靠文件恢复记忆

## 三、固化模板：每次新问题的解决流程

### 问题解决 SOP

```
1. 📊 代入公式
   - 这个问题涉及 APEX 的哪个维度？
   - 当前该维度的分数是多少？
   - 瓶颈在哪？

2. 🔍 三层解法
   - 第一层：我现在就能做的（不需要授权）
   - 第二层：需要授权但可行的
   - 第三层：长期架构级的

3. 🛠 执行
   - 先做第一层，能解决就停
   - 不能解决 → 评估第二层 → 请示授权
   - 记录经验 → 固化到 MD

4. 📝 固化
   - 写入 memory/YYYY-MM-DD.md
   - 更新 MEMORY.md（如果重要）
   - 更新本手册（如果有新发现）
```

### Agent 叠加超能力清单

| 能力 | 怎么用 | 场景 |
|------|--------|------|
| Web 搜索 | mimo_web_search | 实时信息、技术方案 |
| 代码执行 | exec + python | 数据分析、验证推理 |
| 文件操作 | read/write/edit | 知识管理、报告生成 |
| 浏览器 | browser | 复杂网页交互 |
| 子代理 | sessions_spawn | 并行处理多任务 |
| 记忆系统 | memory_store/recall | 跨 session 持久化 |
| 飞书集成 | feishu_* | 文档、表格、知识库 |
