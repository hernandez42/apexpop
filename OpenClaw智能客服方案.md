## 二、核心模块设计

### 2.1 意图识别

在 SKILL.md 中定义分类规则，用户提问时先判断意图：

| 意图类型 | 处理方式 |
|---------|---------|
| FAQ 类 | 检索知识库直接回复 |
| 查询类（订单/物流/账户） | 调 API 查询 |
| 投诉建议 | 创建工单 |
| 闲聊类 | 简短友好回复 |
| 无法处理 | 转人工 |

### 2.2 知识库系统

**数据源：**
- 飞书知识库文档（产品文档、FAQ、操作手册）
- 历史客服对话记录（脱敏后导入）
- 官网帮助中心内容

**检索流程：**
```
用户提问
  ↓
关键词 + 向量混合检索（memory_recall）
  ↓
找到相似度 > 80% 的答案 → 直接回复
找到 60%-80% 的答案 → 参考后组织语言回复
低于 60% → 标记为未解决问题
```

**知识库维护：**
- 定期从飞书文档同步到向量库
- 新问题积累后自动生成 FAQ
- 管理员审核后入库

### 2.3 多轮对话管理

OpenClaw 的 session 系统天然支持多轮对话，每个用户一个独立 session，上下文自动保持。

```
Session A（用户1）
├── 消息1: "你们的退货政策是什么？"
├── 回复1: "7天无理由退货..."
├── 消息2: "运费谁出？"          ← 上下文延续
└── 回复2: "质量问题我们承担..."

Session B（用户2）← 独立会话，互不干扰
```

### 2.4 人工转接

**触发条件：**
- 用户明确要求转人工
- 连续 3 次无法理解用户问题
- 涉及金额/退款/法律等敏感问题
- 用户情绪激动（关键词检测）

**转接方式：**
1. AI 检测到需要转人工
2. 创建飞书文档，包含：用户问题摘要、已尝试的解决方案、对话历史
3. @客服人员，通知转接

**Step 2.2：SKILL.md 核心内容**
```markdown
# Customer Service Skill

## 触发条件
当用户消息包含产品/订单/退款/投诉等客服相关关键词时激活。

## 处理流程
1. 意图分类（FAQ / 查询 / 投诉 / 转人工）
2. 知识库检索（memory_recall）
3. 生成回复（基于检索结果，禁止自由发挥）
4. 质量检查（是否有不确定的内容）
5. 发送回复

## 回复规范
- 语气：友好、专业、简洁
- 长度：不超过300字，重点在前
- 格式：先结论后详情
- 不确定时："我帮您确认一下，稍后回复您"

## 禁止行为
- 编造产品信息
- 承诺超出政策范围的内容
- 泄露其他客户信息
- 讨论竞品
```

**Step 2.3：回复模板**

FAQ 回复模板：
> 您好！关于您问的[问题]，[直接答案]。如需更多帮助，随时告诉我 😊

## 四、关键配置

### 4.1 多模型策略

```json
{
  "models": {
    "providers": {
      "xiaomi": {
        "models": [{
          "id": "mimo-v2-omni",
          "name": "MiMo-Omni（日常问答）"
        }]
      },
      "xiaomi-pro": {
        "models": [{
          "id": "mimo-v2-pro",
          "name": "MiMo-Pro（复杂问题）"
        }]
      }
    }
  }
}
```

路由策略：
- FAQ 类 → MiMo-Omni（免费/便宜）
- 投诉/复杂推理 → MiMo-Pro（更准）
- 深夜低峰 → 本地小模型（省钱）

### 4.2 自动化配置

```json
{
  "cron": [
    {
      "label": "cs-daily-report",
      "schedule": "0 18 * * *",
      "task": "统计今日客服数据：接待量、解决率、未解决问题，生成日报发到群"
    },
    {
      "label": "cs-kb-sync",
      "schedule": "0 3 * * *",
      "task": "同步飞书知识库最新文档到向量数据库"
    }
  ]
}
```

### 4.3 安全边界

- 绝不泄露客户个人信息（姓名、地址、手机号）
- 退款金额超过 500 元必须转人工
- 不回答竞品相关问题
- 不确定的答案说"我帮您确认"而不是猜测
- 用户提供密码/验证码时立即拒绝并提醒安全
