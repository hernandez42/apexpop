# Agent vs 自动化脚本：成本与准确率分析

> 适用场景：高并发、流程固定、输入输出可控的任务

---

## 一、成本对比

### 1.1 成本结构

| 成本项 | 自动化脚本 | Agent |
|--------|-----------|-------|
| 单次执行成本 | 几乎为 0（CPU + 内存） | 每次调用 LLM API，按 token 计费 |
| 开发成本 | 高（需穷举所有分支逻辑） | 低（描述规则即可） |
| 维护成本 | 高（需求变→改代码） | 低（改 prompt 即可） |
| 扩展成本 | 高（新场景→新代码） | 低（加描述就行） |

### 1.2 量化算账

假设日均 10000 次咨询，每次平均 2000 tokens：

| | 自动化脚本 | Agent（MiMo-Omni 免费） | Agent（MiMo-Pro $1/M tokens） |
|---|-----------|----------------|-----------------|
| 日 Token 消耗 | 0 | 2000万 tokens | 2000万 tokens |
| 日 API 成本 | $0 | $0（当前免费） | $20 |
| 月运行成本 | $0 | $0 | $600 |
| 开发人力 | 2人月 ≈ ¥4万 | 0.5人月 ≈ ¥1万 | 0.5人月 ≈ ¥1万 |
| 月维护人力 | 0.5人 | 0.1人 | 0.1人 |

**结论：高并发固定流程场景，自动化脚本的运行成本碾压 Agent。**

---

## 二、准确率对比

### 2.1 各场景准确率

| 场景 | 自动化脚本 | Agent |
|------|-----------|-------|
| 标准输入 → 标准输出 | 99.9%（规则确定） | 95-98%（有概率出错） |
| 边界情况 | 需要穷举，漏了就出错 | 能泛化，但可能过度解读 |
| 输入异常 | 报错/走异常分支 | 可能硬猜一个答案 |
| 一致性 | 100% 一致 | 同样的输入可能输出不同 |

### 2.2 实例对比：查询订单状态

输入："订单号20260322001的状态是什么"

| | 自动化脚本 | Agent |
|---|-----------|-------|
| 正确率 | 99.9% | 97% |
| 响应时间 | 50ms | 500-2000ms |
| 单次成本 | $0.0001 | $0.002 |
| 理解自然语言输入 | ❌ 需要精确订单号 | ✅ 能理解模糊描述 |

---

## 三、决策矩阵

| 任务特征 | 推荐方案 |
|---------|---------|
| 输入格式固定 + 输出固定 + 高并发 | 自动化脚本 |
| 输入多变 + 逻辑复杂 + 低并发 | Agent |
| 大部分固定 + 少量例外 | 脚本为主 + Agent 兜底 |
| 需要理解自然语言 | Agent |
| 毫秒级响应要求 | 自动化脚本 |

---

## 四、最优方案：混合架构

```
用户提问
  ↓
意图识别（轻量规则匹配，非 LLM）
  ↓
┌─────────────────┬─────────────────┐
│ 固定流程任务     │ 非固定任务       │
│                 │                 │
│ 查订单 → 脚本    │ 投诉 → Agent    │
│ 查物流 → 脚本    │ 咨询 → Agent    │
│ 改地址 → 脚本    │ 闲聊 → Agent    │
│                 │                 │
│ 成本: $0/次     │ 成本: $0.002/次  │
│ 准确率: 99.9%   │ 准确率: 95%     │
└─────────────────┴─────────────────┘
```

实际客服场景中，约 60-70% 是固定流程查询，用脚本处理；剩下 30-40% 需要理解力，用 Agent。混合方案比纯 Agent 省 60-70% 成本，比纯脚本多覆盖 30% 场景。

## 五、总结

| 维度 | 自动化脚本胜 | Agent 胜 |
|------|------------|---------|
| 运行成本 | ✅ | |
| 开发效率 | | ✅ |
| 维护成本 | | ✅ |
| 准确率（固定任务） | ✅ | |
| 灵活性 | | ✅ |
| 响应速度 | ✅ | |
| 可解释性 | ✅ | |
| 处理异常情况 | | ✅ |

**核心原则：能用脚本的用脚本，脚本搞不动的才上 Agent。Agent 是瑞士军刀，但能用螺丝刀拧的螺丝别上瑞士军刀。**
